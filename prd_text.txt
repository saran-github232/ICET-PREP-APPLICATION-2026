1. Product Overview
1.1 Purpose
A dedicated, full-featured web-based mock examination portal for students preparing for AP
ICET 2026 (Andhra Pradesh Integrated Common Entrance Test). The portal simulates the real
exam environment, provides AI-powered answers and explanations via Google Gemini, and
includes a curated study guide to help students prepare holistically.
1.2 Core Value Proposition
Authentic exam simulation with timed, section-wise tests
AI-generated motivational quotes on every visit (Gemini)
AI-powered answer keys with step-by-step explanations (Gemini)
21 shift-wise question papers + 2 full Grand Mock tests
Complete APICET previous year papers with structured navigation
Section-ﬁltered question palette (just like the real exam)
Zero backend required — fully client-side with PDF + Gemini
1.3 Target Audience
Students appearing for AP ICET 2026; coaching institutes; self-study aspirants in Andhra
Pradesh and Telangana.
2. Goals & Success Metrics
Goal Success Metric
Students complete at least one mock test ≥ 80% session completion rate
AI explanations load within 3 seconds Gemini API response < 3s per question
Motivational quote refreshes on every visit 100% new quote per page load/refresh
All 21 PDFs load correctly 0 broken paper links
Mock test timer matches real exam Timer accurate ±1 second
Goal Success Metric
Mobile usability Functional on 360px minimum width
3. User Personas
Persona 1 — Priya (Self-Study Aspirant)
Preparing from home, limited coaching access
Needs: Clear guidance, timed practice, Telugu-free papers
Goal: Score 90+ to get into a top MBA college
Persona 2 — Ravi (Last-Minute Crammer)
2 weeks before exam, wants quick mock tests
Needs: Full-length timed tests, instant answer keys
Goal: Identify weak areas quickly
Persona 3 — Coaching Instructor
Wants to share this portal with 30+ students
Needs: Reliable UI, consistent content across sessions
Goal: Replace printed papers with digital practice

7. PDF Content & Naming Convention
7.1 The 21 Shift-wise Papers
User will supply 21 PDFs
Button title = ﬁlename without .pdf, spaces replacing underscores
Example: ICET_2024_Shift1_FN.pdf → Button: ICET 2024 Shift1 FN
7.2 Telugu Language Filter
When extracting questions from PDFs:
Remove all Telugu-script lines, paragraphs, or options
Keep only English-language question text and options
If a question has no English option text at all → skip it
If only some options are Telugu → keep only English options, mark the Telugu ones as
unavailable
7.3 Business & Computer Terminology PDF
Question generation format alternates:
Odd questions: "What is the meaning of [TERM]?" → 4 deﬁnition options
Even questions: "Which term best describes [DEFINITION]?" → 4 term options
Source: This speciﬁc PDF content only
7.4 APICET Previous Year Solved Papers
PDF is large — divide by year into logical groupings
Set 1 and Set 2 division point to be conﬁrmed once PDF is provided
Each year's paper has: Shift indicator, full solutions, AI explanation layer
8. Gemini AI Integration Spec
8.1 Model
Use: gemini-2.0-flash (fast, cost-eﬀective, suﬃcient quality)
Fallback: gemini-1.5-flash
8.2 API Calls Summary
Feature Prompt type Frequency
Motivational Quote Single-turn Every page load
Mock Test Generation (per section) Single-turn, JSON output Once per mock (cached)
Per-question Explanation Single-turn, JSON On-demand per question
Business Terminology Questions Single-turn, JSON array Once (cached)
8.3 Response Format for Questions (JSON Schema)
json
8.4 Error Handling
API key missing → redirect to key input modal
Rate limit (429) → show toast: "Generating too fast. Retrying in 5 seconds..."
Network failure → fallback to cached content if available
Malformed JSON response → retry once; if fails again, show error with retry button
8.5 Caching Strategy
Mock tests stored in localStorage under icet_mock_1, icet_mock_2
Explanations cached individually: explanation_[paperName]_[questionId]
Quote cached per session only (not persisted)

14. Accessibility & Responsiveness
14.1 Accessibility
All interactive elements keyboard-navigable (Tab + Enter)
ARIA labels on palette boxes and option rows
Color is never the ONLY indicator (always paired with text or icon)
Timer announced by screen reader at 10-min and 5-min mark
Minimum contrast ratio: 4.5:1 for body text
Not visited   = id NOT in visited
Not answered  = id in visited, NOT in answers
Answered      = id in answers, NOT in flagged
Review only   = id NOT in answers, id in flagged
Answered+Rev  = id in answers, id in flagged
